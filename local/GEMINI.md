## Обзор проекта

Это веб-сайт, созданный на фреймворке [Astro](https://astro.build/) версии **^5.16.6**. По-видимому, это одностраничный маркетинговый сайт для компании под названием "Kibermatika".

Проект использует [Sass (SCSS)](https://sass-lang.com/) для стилизации, с модульной структурой, где стили разделены на компоненты, макеты и общие утилиты.

Основная страница (`index.astro`) состоит из нескольких отдельных секций, каждая из которых управляется как отдельный компонент `.astro` (например, `Hero.astro`, `Carousel.astro`, `Tickets.astro`). Это указывает на компонентно-ориентированную архитектуру.

## Сборка и запуск

Проект использует `npm` в качестве менеджера пакетов.

1.  **Установка зависимостей:**
    ```bash
    npm install
    ```

2.  **Запуск сервера для разработки:**
    Запускает локальный сервер для разработки с автоматическим обновлением по адресу `http://localhost:4.3.2.1`.
    ```bash
    npm run dev
    ```

3.  **Сборка для продакшена:**
    Компилирует приложение для использования в продакшене. Результат будет находиться в каталоге `dist/`.
    ```bash
    npm run build
    ```

4.  **Предпросмотр продакшн-сборки:**
    Запускает локальный сервер для предпросмотра продакшн-сборки.
    ```bash
    npm run preview
    ```

## Соглашения по разработке

*   **Анимации:** Для анимации блоков будет использоваться библиотека GSAP. Документация по установке: [https://gsap.com/docs/v3/Installation/](https://gsap.com/docs/v3/Installation/)
*   **Компонентный подход:** Пользовательский интерфейс построен из небольших, переиспользуемых компонентов `.astro`, расположенных в `src/components/`.
*   **Модульный SCSS:** Стили организованы с использованием системы "partials" SCSS. Основной точкой входа является `src/styles/main.scss`, который импортирует другие файлы `.scss`. Эта структура способствует поддерживаемости и разделению ответственностей.
*   **Структура файлов:** Каталог `src/` хорошо организован:
    *   `assets/`: Статические ресурсы, такие как изображения и шрифты.
    *   `components/`: Переиспользуемые компоненты пользовательского интерфейса. Разделены на `sections/` для более крупных компонентов страниц.
    *   `layouts/`: Обертки для основных макетов страниц.
    *   `pages/`: Представляют маршруты (страницы) веб-сайта.
    *   `styles/`: Содержит все таблицы стилей SCSS, организованные в `common`, `layout`, `mixins`, и `modules`.
